Real-Time Embedded Systems Programming

Module code: CKRE140

School:- Ryerson University
Year:- 2013

Course Description:-

This course will introduce and explore the programming language and operating system facilities essential to the implementation of real-time, reactive, embedded, robotic and/or networked systems. It will provide practical knowledge of the system issues which must be addressed from design and programming perspective. Differentiated issues arising from design, concurrent, reactive, safety-critical and embedded systems will be explored in depth. Simple hands-on development is facilitated with an embedded system development kit.

Aims-

  • In this course, participants are introduced various building blocks and underlying scientific and engineering principles behind embedded real-time systems. Topics will include real-time system characteristics, time constraints, task scheduling methods, and distributed real-time systems. Participants will examine methods and best practices in implementing real-time systems used in industry today. Through practical hands-on projects, participants will experiment with hardware and software implementation of real-time embedded systems.

    • Learning Outcomes:-

  • Master the various concepts of time that arise in real-time systems.
  • Perform simple performance and scheduling analysis to demonstrate that a system successfully meets real-time constraints.
  • Describe the design and implementation of systems that support real-time applications.
  • Design, construct and analyze a small, concurrent, reactive, real-time system using the recommended hardware kits.
  • Explain problems and constraints with real-time embedded systems
  • Implement an embedded systems that is capable of reading sensor input, logging the data and presenting it and Explain the role of transfer functions and their relationship with the input and output of a system
  • Describe the concept of a PID control loop and Implement a real-time system using a PD controller
  • Identify the timing constraints of different real-time systems and compare and contrast different embedded architecture designs that meet requirements of various real-time systems
  • Describe concepts of servo motors and servo controllers and understand quadrature encoder output
  • Acquire programming techniques on external interrupts and reading quadrature encoder
  • Implement a real-time embedded system using PID control loop
  • Compare and contrast speed controllers with position controllers and describe methods and techniques of designing a distributed real-time system
  • Apply principles of real-time systems (multitasking, scheduling, synchronization)

    Contents:-

    Topics Covered-
  • Introduction Real-Time Embedded System. Topics Covered are - Navigating through your online course What are Real-Time systems and their characteristics? Design Constraints of Real-Time Embedded System
  • Hardware Interface. Topics covered are - Sensor Input, PWM Output, Transfer Function
  • Introduction to PID Controller. Topics covered are - Why learn PID controllers, PID Controller Example Application, PID Controller Tuning Strategy
  • Timing Constraints: Embedded Architecture. Topic Covered: Segway PT Case Study
  • Servo Motor and Quadrature Encoder. Topic covered: Servo Motor, R/C Servo Motor, Quadrature Encoders
  • Programming External Interrupts. Topics covered: Why Use Interrupts? and External Interrupt with Atmega328p
  • Clocks, Timers, and Periodic Execution. Topics covered: Keeping Time, Timers with Atmega328p
  • Full PID Controller. Topics covered: The 'I' in PID, Fine tuning the PID controller, Practice exercise on tuning a full PID loop
  • Speed Control vs. Position Control. Topics covered: Motor Control: constant speed, Measuring speed using encoders, Practice exercise on using reading encoders
  • Multitasking, Scheduling, Synchronization. Topics covered: Multitasking, Task Scheduling, Synchronization
  • Distributed Real-Time Systems. Topics Covered: Multiprocessors, Serial communications, Task Scheduling on Multiprocessors
